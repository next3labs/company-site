# Windows File Server Access Solution for Lights & Shadows **Date:** 24 February 2026 --- ## The Problem Lights & Shadows has **Windows machines in China** that need to access their **file server in Singapore**. The server is behind a **Fortinet firewall**. ### Current Challenges: - 🔴 High latency (150-200ms) between China and Singapore - 🔴 China's Great Firewall causes unstable connections - 🔴 Fortinet requires careful port configuration - 🔴 Windows SMB protocol is sensitive to latency --- ## Our Solution: Alibaba Cloud Global Accelerator ### Architecture ``` ┌─────────────┐ ┌──────────────────────┐ ┌─────────────┐ │ CHINA │ │ ALIBABA CLOUD │ │ SINGAPORE │ │ │ │ GLOBAL ACCELERATOR │ │ │ │ ┌───────┐ │ │ │ │ ┌───────┐ │ │ │Windows│──────▶│ China → Singapore │──────▶│Fortinet│ │ │ │ PC │ │ │ (Optimized Route) │ │ │ FW │ │ │ └───────┘ │ │ │ │ └──┬────┘ │ │ │ └──────────────────────┘ │ │ │ │ │ │ ▼ │ │ │ │ ┌───────┐ │ │ │ │ │ File │ │ │ │ │ │Server │ │ │ │ │ └───────┘ │ └─────────────┘ └─────────────┘ ``` ### How It Works 1. **Connect:** User in China connects to nearest Alibaba Cloud GA point 2. **Route:** Traffic flows through Alibaba's global backbone (not public internet) 3. **Terminate:** Traffic arrives at Singapore through Fortinet 4. **Access:** Windows SMB connects to file server as if on local network ### Why Alibaba Cloud?
